# ВЫВОДЫ ПО ЛАБОРАТОРНОЙ РАБОТЕ №3

## Что делали

Делали кластеризацию клиентов банка — разбивали их на группы по похожим характеристикам.

Почему кластеризацию, а не регрессию или рекомендации:
- Регрессия не подходит — нечего предсказывать (y бинарная, duration известна только после звонка)
- Рекомендации не подходит — нет данных "кто что купил"
- Кластеризация подходит — есть характеристики клиентов, можно разбить на группы

## Что сделали

1. Взяли датасет из ЛР1 и ЛР2 (41176 клиентов, 11 признаков)
2. Попробовали 3 метода:
   - **KMeans** — базовый метод (baseline)
   - **DBSCAN** — находит выбросы
   - **Hierarchical** — строит дерево кластеров

## Сравнение методов

| Метод | Silhouette | Davies-Bouldin | Комментарий |
|-------|------------|----------------|-------------|
| KMeans | 0.17 | 1.75 | Равномерные кластеры |
| **DBSCAN** | **0.31** | 1.21 | **Победитель!** |
| Hierarchical | 0.31 | 0.95 | Один кластер всего 3 точки |

**Победитель: DBSCAN** (Silhouette = 0.31)

## Главный результат

Нашли 4 кластера + шум с **очень разной конверсией**:

| Кластер | Размер | Конверсия | Что за клиенты |
|---------|--------|-----------|----------------|
| -1 (шум) | 41 | 43.9% | Нетипичные клиенты — выбросы |
| 0 | 35541 | 8.8% | Основная масса клиентов |
| 1 | 4105 | 13.0% | Клиенты с историей контактов |
| **2** | **1361** | **65.0%** | **ЛУЧШИЙ СЕГМЕНТ!** |
| 3 | 128 | 50.0% | Маленькая группа с высокой конверсией |

**Главный инсайт:** Кластер 2 — всего 1361 клиент (3.3%), но конверсия 65%! Это в 7 раз выше, чем у основной массы.

## Почему кластер 2 такой крутой?

Это клиенты у которых:
- Есть история контактов с банком
- С ними связывались в прошлых кампаниях
- Они уже знакомы с предложениями

Короче — это "тёплые" клиенты.

## Ещё интересное

- **Шум (-1)** — 41 человек с конверсией 44%. Это нетипичные клиенты, которые не вписываются ни в один кластер, но часто соглашаются. Стоит изучить их отдельно!
- **Кластер 3** — всего 128 человек, но конверсия 50%

## Метрики

- **Silhouette = 0.31** (DBSCAN) — неплохая структура
- **Inertia = 308552** (KMeans) — использовали для выбора K=4
- **ARI = 0.05** — низкий, но это ок — кластеры не должны совпадать с y

## Проблемы

- Кластер 0 слишком большой (35541 из 41176) — 86% данных в одном кластере
- Label Encoding для категорий — не идеально
- PCA объясняет только 28% дисперсии

## Что можно улучшить

- Попробовать другие параметры DBSCAN чтобы разбить кластер 0
- One-Hot Encoding вместо Label
- Создать новые признаки (возрастные группы)
- Изучить отдельно "шум" — там интересные клиенты

## Итог

Разбили клиентов на группы с помощью DBSCAN. Нашли сегмент с конверсией 65% (кластер 2) — это "тёплые" клиенты с историей контактов. Банку надо звонить им в первую очередь.
